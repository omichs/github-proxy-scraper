# 🕵️ GitHub Proxy Scraper

Многопоточный Python-инструмент для сбора бесплатных прокси-серверов из публичных репозиториев GitHub. Автоматически сканирует файлы, удаляет дубликаты, фильтрует приватные IP-адреса и опционально проверяет работоспособность каждого прокси перед сохранением.

---

## ✨ Возможности

- **Параллельное сканирование** — одновременная обработка нескольких репозиториев через `ThreadPoolExecutor`
- **Умный парсинг** — рекурсивное извлечение прокси из файлов `.txt`, `.json` и `.xml`
- **Строгая валидация** — регулярное выражение проверяет корректность октетов IP и диапазон порта `1–65535`
- **Фильтрация публичных IP** — отбрасываются приватные, loopback, link-local и зарезервированные адреса
- **Retry-логика** — автоматические повторные запросы при ошибках `429 / 500 / 502 / 503 / 504` с экспоненциальной задержкой
- **Защита от rate limit** — автоматически ждёт сброса квоты GitHub API при её исчерпании
- **Проверка прокси** — флаг `--check` валидирует каждый прокси через `httpbin.org`
- **Режим дозаписи** — `--append` объединяет новые результаты с существующим файлом вместо перезаписи
- **Полноценный CLI** — все параметры настраиваются через аргументы командной строки

## 📋 Требования

- Python 3.8+

## 🚀 Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/omichs/github-proxy-scraper.git
cd github-proxy-scraper
```

### 2. Установить зависимости

```bash
pip install requests tqdm
```

### 3. Настроить список репозиториев

Отредактируйте `repositories.txt` — добавьте URL репозиториев GitHub, по одному на строку:

```
https://github.com/TheSpeedX/PROXY-List
https://github.com/monosans/proxy-list
https://github.com/ErcinDedeoglu/proxies
```

Строки, начинающиеся с `#`, воспринимаются как комментарии и игнорируются.

### 4. Запустить

```bash
# Базовый запуск
python proxy_scraper.py

# Со всеми параметрами
python proxy_scraper.py \
  --repos repositories.txt \
  --output proxies_output.txt \
  --workers 15 \
  --timeout 10 \
  --check \
  --check-workers 50 \
  --check-timeout 7 \
  --append
```

---

## ⚙️ Аргументы командной строки

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `--repos` | `repositories.txt` | Файл со списком репозиториев |
| `--output` | `proxies_output.txt` | Файл для сохранения прокси |
| `--workers` | `10` | Количество потоков для сканирования |
| `--timeout` | `10` | Таймаут HTTP-запроса (секунды) |
| `--check` | выкл | Проверять прокси перед сохранением |
| `--check-workers` | `50` | Потоки для проверки прокси |
| `--check-timeout` | `7` | Таймаут одной проверки прокси (секунды) |
| `--append` | выкл | Дозаписывать в существующий файл |

## 🔑 GitHub API Token (рекомендуется)

Без токена GitHub API допускает только **60 запросов в час**. С токеном — **5 000 запросов в час**.

```bash
# Linux / macOS
export GITHUB_TOKEN=ghp_your_token_here
python proxy_scraper.py

# Windows (PowerShell)
$env:GITHUB_TOKEN="ghp_your_token_here"
python proxy_scraper.py
```

Как создать токен: [GitHub Docs → Personal Access Tokens](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token)

> Достаточно токена **без каких-либо scopes** — только публичный доступ на чтение.

## 📁 Результат

Собранные прокси сохраняются в `proxies_output.txt` — по одному адресу `IP:PORT` на строку, в отсортированном виде:

```
1.2.3.4:8080
5.6.7.8:3128
...
```
